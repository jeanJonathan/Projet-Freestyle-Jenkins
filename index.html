Jenkins interrogera GitHub pour détecter les changements selon cet intervalle.

Car si Jenkins détecte des modifications depuis la dernière interrogation, il déclenchera automatiquement un nouveau build.

En effet Chaque fois que nous poussons une mise à jour sur GitHub, Jenkins récupère ces mises à jour et déclenche un build automatiquement. Le build pourrait inclure des étapes pour tester votre code, le construire, et le déployer. Cela permet d'assurer que votre code est toujours dans un état "construisible" et que toute nouvelle modification est intégrée rapidement, ce qui est une pratique clé de l'intégration continue.
